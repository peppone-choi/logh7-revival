# Workflow Selection Policy

## Purpose

작업 특성에 따라 사람-only, AI-assisted, AI-native 중 어떤 방식을 선택할지 결정하는 정책.
발표 자료가 아니라 **작업 방식 선택 규칙**이다. 근거 없는 생산성 수치를 쓰지 않는다.

## Decision Matrix

| 작업 유형 | 권장 방식 | 사람 역할 | AI 역할 | 승인 지점 |
|---|---|---|---|---|
| 캐논 판정 (게임 규칙·데이터 승격) | 사람-only (AI는 증거 수집만) | 최종 판정 | CD·매뉴얼·패킷 증거 수집·대조 | 데이터 승격 규칙 (`logh7-roadmap-current.md`) |
| 아키텍처·프로토콜 설계 결정 | Hybrid | 결정·승인 | 대안 비교·영향 분석 | `.ai/decisions.md` ADR |
| 서버 기능 구현 (명세 확정) | AI-native | 계약 승인·diff 확인 | 구현·테스트·문서 동기화 | 계획 승인, 병합 승인 |
| RE·전수 파싱·자산 추출 | AI-native | 결과 샘플 검증 | 반복 분석·카탈로그 생성 | 캐논 승격 시 |
| 테스트 작성·회귀 검증 | AI-native | 기준 승인 | 초안·실행·기록 | 테스트 약화 시 |
| 라이브 QA (원본 클라이언트) | Hybrid | 시각 판정·이상 감지 | 하네스 구동·로그 상관 | run 계획, 서버 재시작 |
| 문서 현행화 | AI-native | 표본 검사 | 갱신·인덱스 정리 | 정책 문서 변경 시 |
| 병합·릴리스·데이터 삭제 | 사람-only | 실행 승인 | 사전 검증 증거 준비 | 항상 |

## Comparison

| 축 | 사람-only | AI-native | 운영상의 의미 |
|---|---|---|---|
| 시간 | 직렬, 사람 가용시간에 종속 | 병렬 위임 가능, 상시 | 긴 전수 작업(RE 스윕·카탈로그)은 AI로 |
| 탐색폭 | 경험 기반 표본 | 전수 탐색 가능 | 단, 잘못된 전제도 전수로 확산됨 — 전제를 사람이 고정 |
| 검증 루프 | 수동, 드묾 | 훅·테스트로 매 편집 | 검증 자동화가 없으면 AI 속도는 위험으로 전환 |
| 실패 패턴 | 누락·피로 | 환각·범위 이탈·허위 완료 (`failure-cases.md`) | 게이트(훅·승인·증거)로 상쇄 |

## Human-Only Required

비즈니스·마일스톤 우선순위, 비가역 아키텍처 결정, 라이브 데이터 삭제, 보안 승인, 라이선스 판단(외부 레포 코드), 주관적 UX·재현 충실도 판정, 비용·위험의 최종 수용, 병합·배포.

## AI-Native Recommended

관련 파일 탐색, 기존 패턴 조사, 반복 구현, 테스트 초안, 정적 분석, 문서 동기화, 변경 요약, 다수 대안 비교, 전수 파싱·카탈로그.

## Hybrid Required

프로토콜 해석(AI 분석 + 라이브 오라클 확인), 라이브 QA, 스키마 변경, 한글화 방식 선택(인코딩 전략).

## Escalation Rules

- AI가 UNKNOWN·NEEDS_HUMAN_CONFIRMATION을 만나면 추측하지 않고 에스컬레이션.
- Blocked-Loop(같은 증상 3회 / 새 증거 없는 조사 2회) 도달 시 블로커 보고서와 함께 에스컬레이션.
- 계약 범위 밖 수정 필요 발견 시 계약 갱신 요청.

## Stop Conditions

승인 게이트 항목 도달, 소유 충돌, 민감 파일 접근 필요, 검증 불가능(도구 부재·P0 게이트 차단) 상태에서의 완료 요구.

## AI-native 주요 실패 패턴 (상세: `failure-cases.md`)

관련 없는 변경 확산 / 존재하지 않는 API·경로 사용 / 테스트 미실행 완료 주장 / 긴 대화에서 제약 누락 / 잘못된 전제의 자동 확산(0x030b 사례) / 두 에이전트 동일 파일 동시 수정.
