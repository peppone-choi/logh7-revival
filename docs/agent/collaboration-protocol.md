# Collaboration Protocol

Claude Code와 Codex(및 서브에이전트)가 같은 저장소에서 충돌 없이 일하기 위한 규칙.
실행 절차는 `lifecycle-collaboration.md`, 소유권 현황은 `.ai/ownership.md`.

## 공통 핵심 규칙 (모든 에이전트)

1. **single-writer-per-file**: 한 파일에는 한 시점에 한 에이전트만 writer. 시작 전 `.ai/ownership.md` 등록, 종료 시 해제.
2. 다른 에이전트 소유 파일은 읽기만 한다. 덮어쓰기 금지.
3. 동일 기능을 병렬 구현하지 않는다. 병렬화는 독립 조사·테스트·리뷰·문서화에만.
4. 브랜치 또는 worktree로 격리한다 (현행 관례: `codex/*` 브랜치 → PR).
5. 결과 전달은 `.ai/handoff.md` — 대화 기록에 의존하지 않는다.
6. 한 에이전트의 **추측을 다른 에이전트가 사실로 승격하지 않는다**. handoff에서 결정(승인됨)과 추측(Inferred)을 구분 표기한다.
7. 완료 보고를 그대로 믿지 않는다 — 검증자는 diff와 테스트 출력을 직접 확인한다.

## Claude 전용: Advisor Strategy와 모델 라우팅

(Codex에는 적용되지 않는다. 근거·이력: 루트 `CLAUDE.md` 하네스 변경 이력)

- 메인 세션(Fable)은 Advisor 전용 — 요구사항 분해·설계 결정·브리프 작성·검증·승인만 직접 하고, 탐색·구현·전수 파싱은 서브에이전트에 위임한다. 긴 파일 읽기·구현 노동을 메인 컨텍스트에서 하지 않는다.
- 라우팅은 Opus 아니면 Haiku (Sonnet 등급 폐지, 2026-07-09):
  - **opus**: 설계 결정(아키텍처·프로토콜·스키마), 다중 가설 근본원인 진단, 여러 파일·계층 추론 구현, 모호한 명세 확정, 최종 판정. 순수 사고는 deep-reasoner(effort max).
  - **haiku**: 그 외 전부 — 명확한 지침의 구현·수정·테스트, 명령 실행·조회·파일 스윕.
  - 위임 시 **model 명시 필수** (생략하면 세션 모델을 상속해 잡무에 최고가 토큰 소모). 애매하면 haiku 먼저.
- 조언자 패턴: 실행자가 막히면 태스크 재위임 대신 판단 질문만 추출해 deep-reasoner에 짧게(400~700토큰) 묻고 그 지시로 재개. 태스크당 3회 이내.
- 브리프에 필수: 파일 경로, 이미 확인한 사실, 왜(의도), 금지 범위(범위 밖 수정·작업트리 리셋·미지시 기능 금지), 완료 테스트.
- effort 기본 low~medium. 어려운 판단만 high, xhigh/max는 크리티컬 단독 판단만.
- 부모 Worker의 하위 에이전트 생성은 메인이 명시한 경우 외 금지.

## 검증 경계 (공통)

- Worker 보고 → 메인/사람이 diff·테스트로 직접 확인 → 승인. 실패는 수정 브리프로 재위임.
- 직접 수정은 위임 오버헤드가 더 큰 사소한 마무리(한두 줄)에만.
