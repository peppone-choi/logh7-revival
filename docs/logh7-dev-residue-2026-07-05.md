## 핵심 발견 요약

1. `thumbs.db`(2003-07-28 오전)와 `_catalog.vix`(2003-07-06~08-23)는 서로 다른 시기의 개발 스냅샷이며, 둘 다 "썸네일엔 있는데 실파일 없음" 케이스는 0건 — 자산 삭제 흔적보다는 **캐시 생성 이후 자산 추가**가 지배적 패턴.
2. `icon_kj` 아이콘 세트는 2003-07-06 단발 시작 → 07-30 대량 배치(전술 아이콘 28종) → 08-19 2차 대량 배치(상태 세부 아이콘) → 08-23 마무리, 총 48일에 걸친 3단계 제작 타임라인으로 재구성됨.
3. `icon_kj\_catalog.vix` 헤더에 개발자 경고 코멘트가 그대로 박제됨: "W:\Gin7\FIX ... 난이도 데이터를 넣지 마세요"(추정) — 해당 폴더를 확정판으로 취급했다는 내부 규율 단서.
4. `models.json`이 참조하는 임베디드 경로로 개발환경 복원: 로컬 `W:\Gin7\CG\` + 사내 공유서버 `\\Daemon\pub\Gin7\CG\`, 함선/민간선/천체/2D전술/이펙트로 분류된 트리, 함선 오브젝트는 `EH/EL/EM/FH` 접두 코드 체계.
5. `Efect`/`Effect` 오타가 실제 배포 자산 경로에 그대로 섞여 남아있어("우메모트" 작업자 트리 하위), 정식 출시본까지 정정되지 않은 사소한 개발 잔재로 확인됨. `kinkyuu_hokyuu.tga`/`kinkyuuhokyuu.tga` 언더스코어 유무 중복 파일도 동일 계열의 명명 잔재.
