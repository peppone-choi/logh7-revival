## G001 세부 단계

> C002 종결 경로 확정(`docs/logh7-loop-state.md` 2026-06-23 블록): 0x0b01 단일 게이트 = 마우스 클릭이 함대/명령 위젯 rect hit → `+0xb00` 발화 + 명령메뉴 rowCount>0.

1. **RE 프리패스**: `FUN_004f5cb0`(명령 메뉴 빌더), `FUN_005015f0`(row hit-test), `FUN_004f93c0`(command factory), `FUN_00581c80`(SelectGrid), `FUN_005737d0`(SendWarpCommand) 매개변수/구조체/위젯 rect를 Ghidra 인덱스로 재확인.
2. **함대 선택 hit-test**: mode2 전략 화면에서 own-fleet 마커/위젯의 rect를 계산/시드할 수 있는 안전 경로(RE 확인된 메모리/와이어링) 구현.
3. **명령 카탈로그 빌더 트리거**: 서버가 0x0305/0x0307 또는 동등 구조를 통해 `FUN_004f5cb0`가 rowCount>0인 명령 메뉴를 populate하도록 데이터 배선.
4. **row 클릭 에뮬레이션/라우팅**: 실제 마우스 클릭 또는 메모리/와이어 단축이 `FUN_005737d0`까지 도달해 `0x0b01`을 발생시킴.
5. **실클 검증**: trace + 스크린샷 + EXE SHA 복구.
