# 0x0317 ResponseInformationGrid 셀렉터 레버 — C002 mode2 라이브-prep (2026-06-26)

## 요약
캐논 서버는 0x0317(ResponseInformationGrid)을 발신하지 않아(login-session은 0x0315 grid만) 클라
`clientBase+0x35f358` grid dword가 미설정 → mode 분기가 mode2 고정 추정. 이 레버는 월드 도달 후 0x0317을
지정 grid u32로 1회 푸시해 셀렉터 바이트(byte[2] = `+0x35f35a`)를 제어, mode 분기를 라이브로 관측한다.
**byte-correct emit + 오라클 테스트만 보장. 실제 mode2 유발은 라이브 probe로 확인(단정 금지).**

## RE 확정 (redex over .omo/ghidra/export/G7MTClient)
- **수신(파서) FUN_004ba2b0 `case 0x317`**: `*(undefined4 *)(param_1 + 0x35f358) = *param_3;`
  → inner payload 첫 dword를 `clientBase+0x35f358`에 LE u32로 그대로 복사(단일 dword, SIZE 4).
- **셀렉터(mode 분기) FUN_004b68f0 `@0x35f35a`**:
  ```
  iVar7 = 2; local_20 = 2;
  if (param_1[0x35f35a] != '\0') { iVar7 = 1; local_20 = 1; }
  ```
  `0x35f35a = 0x35f358 + 2` = grid dword의 **byte[2]**. byte!=0 → iVar7=1, byte==0 → iVar7=2.
- 따라서 **셀렉터 byte = `(grid >> 16) & 0xff`** (LE u32 byte[2]).

## 서버 빌더 (byte-correct, 무변경 — 이미 존재)
`src/server/logh7-info-records-static.mjs` `buildInformationGridInner({ grid })`:
- `inner.subarray(6).writeUInt32LE(clampU32(grid), 0x00)` → body[0..3] = grid LE u32.
- body byte[2] = `(grid>>16)&0xff` = 클라 0x35f35a 셀렉터. **파서와 정확 일치.**

## 레버 게이트 (off-default, 무회귀)
`src/server/logh7-login-session.mjs`:
- `LOGH_GRID_SELECTOR_PROBE=1` — grid-enter 시 0x0317을 `postloadExtras`에 1회 추가(즉시, deferred 아님 —
  single dword라 렌더 stall 위험 없음). 두 grid-enter 경로 모두 배선: SS_REQ_MESSENGER_STAT 경로 +
  `0x0f06` 경로.
- `LOGH_GRID_SELECTOR_VALUE`(u32, 기본 0) — 푸시할 grid dword. 예: `0x00010000` → byte[2]=1 → iVar7=1.
- 기존 deferred probe(battle/fleet-move/state-transition)와 **무충돌**: `deferredBattleInners` 필드를
  쓰지 않고 `postloadExtras`(즉시 extraInners)에만 push.

## 라이브 실험 설계 (probe로 0x35f35a / mode 관측)
1. `LOGH_GRID_SELECTOR_PROBE=1 LOGH_GRID_SELECTOR_VALUE=0x00010000` (+ `LOGH_GRID_ENTER=1`)로 서버 기동.
2. logh7-live 스킬로 실클라(canonical SHA) 월드진입(autologin 금지=실로그인, 스플래시 ~35초 포그라운드).
3. ui_explorer read로 `clientBase+0x35f358` dword(=0x00010000 기대) 및 `+0x35f35a` byte(=1 기대) 확인 —
   0x0317 수신·복사 도달 검증.
4. FUN_004b68f0 진입 시 iVar7(=1 vs 2) 관측 → 셀렉터가 실제 mode 분기를 뒤집는지 라이브 확정/반증.

5. value=0(셀렉터 0)과 value!=0 A/B 대조로 인과 격리. 결과를 old client-state journal removed during 2026-07-03 cleanup; fresh current evidence path에 기록.
