## FUN_005737d0 — SendWarpCommand (0x0b01 dispatch 말단)
- 호출규약: `__thiscall(int param_1=this, uint param_2=mode/branch idx)`. 반환 undefined4(성공 플래그).
- purpose: 함대 워프/이동 명령을 목적지 grid로 확정·송신하는 명령 노드의 실행 step. `s_TARGET_SELECT_S_BASE`(0x78bb84), `s_TARGET_BASE_GRID`(0x78bf6c) 디버그 라벨 emit → 타깃 선택→목적지 grid 산출.
- 핵심 분기: `iVar5 = FUN_004f8ee0(this)` 결과로 mode(5/6=별도 처리, else=타깃순회). `this+0x28`(==0)일 때 TARGET_SELECT/TARGET_BASE_GRID 컬렉션을 순회(`this+0x10`=리스트 base, `this+4`=인덱스)하여 목적지 노드(local_28)를 찾고 `+0xc`로 grid 추출.
- 피호출자: FUN_004b5bb0, FUN_004f8ee0, FUN_00575510(타깃 매칭), FUN_00600d3a, FUN_004fde60(상태 push), FUN_004b4a00. **실제 와이어 송신은 본 함수 vtable 형제(FUN_004b48d0→FUN_004b78a0(1,0x3b=GRID))** — 본 함수는 타깃/grid 확정 후 디스패치 트리거.
- 호출자: redex `calls`로 직접 호출자 0(자기 자신만) = **vtable 간접 호출**(노드 vtable PTR_FUN_00676aec+8, FUN_004f93c0이 인스턴스화한 노드를 task runner가 vtable로 invoke). 직접 xref 없음이 정상.
- DAT: DAT_021aa888(컨텍스트), s_TARGET_SELECT_S_BASE_0078bb84, s_TARGET_BASE_GRID_0078bf6c, DAT_00779460, DAT_00c9e2f8(상태 게이트 — 0이면 selection idx 리셋).
- C002 역할: 체인 말단. 여기 도달하면 0x0b01 송신. 도달 자체가 상류(메뉴 row 클릭) 미구동으로 차단됨.

## FUN_004f93c0 — SendWarpCommand 인스턴스화 (명령 노드 생성)
- 호출규약: `__thiscall(char *param_1=this(명령위젯), int param_2=cmdIdx, undefined4 param_3=context/mode)`. 반환 undefined1(성공).
- purpose: 선택된 명령 row(param_2=cmdIdx)에 대해 vtable 슬롯 `*(this + param_2*4 + 0x1c)`의 팩토리를 호출해 명령 노드(iVar3)를 생성하고 `iVar3+0x28=cmdIdx`, `iVar3+0x1c=DAT_021aa888`, `iVar3+0x20=param_3`를 기록 후 명령 큐(this+0x10 링크드 리스트)에 연결. 곧 task runner가 그 노드의 SendWarpCommand step을 실행.
- 게이트: `*this != 0`(위젯 활성) && `*(this+0x18)==0`(미점유). 팩토리 슬롯이 0이면 실패(예외 경로 FUN_004fde60).
- 피호출자: vtable factory `(*(this+param_2*4+0x1c))()`, FUN_004c8d30, FUN_00517db0, FUN_00648d42(리스트 노드 alloc), FUN_004fde60.
- 호출자: **FUN_004f58c0**(명령 메뉴 row 클릭 hit 시).
- DAT: DAT_021aa888, PTR_DAT_007b3608, DAT_00779460, DAT_00c9e2f8.
- C002 역할: 메뉴 row 클릭→명령 노드 생성. param_2*4+0x1c 팩토리 테이블 = 클라 내장 명령 카탈로그 dispatch.

## FUN_004f58c0 — 명령 메뉴 row 클릭 hit-test → dispatch
- 호출규약: `__thiscall(int *param_1=this(명령메뉴 위젯), int *param_2=outFlag)`. 반환 void.
- purpose: 명령 메뉴 패널의 각 row 위젯(this+0xc 배열, 개수 this[0xd4])을 순회하며 `FUN_005015f0(2, rowWidget, ...)` 좌클릭 hit-test. hit한 row 인덱스(iVar4)로 `this[0xd5]=selectedIdx` 기록, 0..0x17 범위면 `this[0xd6]`(카테고리)·FUN_004c8700 테이블에서 명령 코드 uVar6 산출 → **FUN_004f93c0(uVar6, this[0xd6])** 호출(노드 인스턴스화) + FUN_005088e0(하이라이트), `*param_2=1`.
- 게이트: `this[0xd4]`(rowCount)>0 필요. 끝에서 `DAT_00c9e2f8==0`이면 selectedIdx 리셋.
- 피호출자: FUN_005015f0(2)(클릭 hit-test), FUN_004c8700(명령코드 테이블), FUN_004f93c0, FUN_005088e0.
- 호출자: **FUN_004fd100(case1, StrategySequence 메인 분기)**.
- DAT: DAT_00c9e2f8.
- C002 역할: **진짜 dispatch 트리거 = 명령 메뉴 row 클릭**(별/grid 셀 클릭 아님). rowCount(this[0xd4])>0이 선결 — 메뉴가 안 채워지면 순회 0회.

## FUN_004f6680 — unit-list 패널(0x67) populate
- 호출규약: `__thiscall(int *param_1=this(unit-list 패널), int param_2=tab/category 1..3)`. 반환 undefined4(0/1).
- purpose: unit-list(officer/함대) 패널의 탭(param_2)을 선택하고 row 위젯들을 채움/배치. **선결 게이트 `FUN_0050cf40(0x67)!=0`** — 위젯 0x67(unit-list 패널 객체)이 존재해야 진행, 없으면 즉시 0 반환. param_2 1~3 범위, 직전 탭과 같으면 1 반환(noop).
- 본문: row 슬롯 `param_1+2`부터 0x40(64)개 순회하며 FUN_00502eb0/ee0/24e0로 가시성 토글. `&DAT_0066f130 + tab*0x208` 테이블로 row 데이터 매핑, 각 row를 FUN_00502f40로 좌표 배치, 합계(param_2 누적)>0이면 스크롤바 갱신(FUN_00502940).
- 피호출자: FUN_0050cf40(0x67)(위젯 조회 — **핵심 게이트**), FUN_004ea610, FUN_00502ea0/24b0/2eb0/2ee0/24e0/2980/2f40/2940.
- 호출자: FUN_004f6040(빌더), FUN_004f68f0(officer 데이터), FUN_004fd7a0(catGate writer).
- DAT: DAT_0066f130/0066f134(row 테이블, stride 0x208), DAT_007c1b4c, DAT_0066e* (레이아웃 float).
- C002 역할: 함대선택(레이어 4)의 대상 row를 제공. **상류 근본 = 0x67 위젯 자체가 빌더 FUN_004f6040 미실행으로 미생성** → FUN_0050cf40(0x67)==0 → populate 즉시 bail(또는 catGate 강제 시 garbage 역참조 크래시, live19 @0x687fa80과 정합).

## C002 종결 체인에서의 역할 (패널구성 → row클릭 → dispatch → 0x0b01)
```
[상류 근본] FUN_004f6040 (0x67 unit-list 패널 빌더) — autologin/revival 월드서 미실행
   └ FUN_004f6680: FUN_0050cf40(0x67)==0 게이트로 populate 불가 (행 0)
        └ 함대선택 row 클릭(FUN_005015f0(2) hit, +0xb00 latch) — 위젯 미존재로 발화 불가
             └ 명령메뉴 build (rowCount this[0xd4]>0) — 선택 선결, 현재 rowCount=0
                  └ FUN_004f58c0: row 클릭 hit → cmdIdx 산출
                       └ FUN_004f93c0: 팩토리 슬롯 호출 → 명령 노드 인스턴스화
                            └ FUN_005737d0(SendWarpCommand): 타깃/grid 확정 → 0x0b01 송신
```
게이트 조건 요약:
- FUN_004f6680: `FUN_0050cf40(0x67)!=0` (패널 위젯 존재) + tab 1~3.
- FUN_004f58c0: `this[0xd4]`(rowCount)>0 (메뉴 채워짐).
- FUN_004f93c0: `*this!=0` && `*(this+0x18)==0` && 팩토리 슬롯 `*(this+cmdIdx*4+0x1c)!=0`.
- FUN_005737d0: `this+0x28==0` 분기서 타깃 컬렉션에 후보 존재.

## 미해결
- FUN_004f6040(0x67 빌더)의 자연 트리거 경로(FUN_0054e570→FUN_004ff3c0→FUN_004fc4e0→FUN_004f6040)가 autologin 월드서 왜 미실행인지 — 씬-셋업 진입 조건 RE 미완(B조 후보).
- DAT_0066f130 row 테이블(stride 0x208)이 클라 내장 정적인지 서버 0x0323/0x0325 데이터 의존인지 미확정.
- FUN_004f93c0의 팩토리 vtable 테이블(this+0x1c base) 실주소·명령 카탈로그 항목 수 미열거.
