# 진영명(faction name, constmsg group 1) 소비처 확정 — 2026-06-29 (P0-07)

목표(P0-07): "group-1 진영명 접근자 xref → 맵/패널 소비처 확정". RE-확정만 기록. 추측 승격 없음.
도구: redex (`.omo/ghidra/export/G7MTClient`), imagebase 0x400000.

## 1. 접근자

- constmsg 텍스트 접근자: **`FUN_00522010(group, subId)`** (마스터 로더 `FUN_00521dc0`가 적재).
- 진영명 = **group 1**. subId 규약(`docs/logh7-character-creation-wire.md` §): `2=제국`, `3=동맹`.
- 시스템(성계)명은 group `0x18`(`FUN_004c8c90 → FUN_00522010(0x18, byte0)`), 출신/혈통은 group `0xf`,
  작위/직무는 group 3·5 — 모두 같은 접근자의 다른 group이다.

## 2. group-1 진영명 직접 소비처 (literal `FUN_00522010(1, …)` xref, 전수)

redex `calls 0x00522010`로 전 호출자 39개를 얻고, 그중 **group 인자가 리터럴 1**인 함수만 필터:
`grep "FUN_00522010\(1,"` → **정확히 2개**.

| 함수 | 호출 | 화면/역할 | subId 출처 | 분류 |
|---|---|---|---|---|
| `FUN_00597b20` | `FUN_00522010(1, _DAT_02227f68 & 0xff)` | 캐릭터 정보/프로필 패널 (window2 텍스트 슬롯 `FUN_00502780(2,n)`+`FUN_00503560`) | 패킹된 char-state 전역 `_DAT_02227f68`의 **low byte = power/진영** | **패널(캐릭터 정보)** |
| `FUN_0059df00` | `FUN_00522010(1, 2)` (외 1) | 게임메뉴/윈도우 빌더 (`data/image/gamemenu/menu_par`, `window/window_par` 아틀라스) | **리터럴 2(=제국)** 고정 라벨 | **메뉴/윈도우 고정 라벨** |

- `FUN_00597b20`은 같은 전역 `_DAT_02227f68`에서 진영(low byte, group1)·출신(high byte, group 0xf)을
  함께 읽고, 작위(group 3)·계급접미(group 5)도 같은 패널에 채운다. 즉 진영명은 **선택/로컬 캐릭터의
  프로필 텍스트**로만 group-1을 쓴다(맵 유닛/성계당 렌더 아님).
- `FUN_0059df00`은 메뉴 아틀라스 기반 빌더로 진영명을 **고정 라벨**로 박는다(런타임 진영값 아님).

## 3. 맵/패널의 "진영 표시"는 group-1 이름을 쓰지 않는다 (정정/확정)

| 표시 | 출처 함수 | 메커니즘 | group-1 사용? |
|---|---|---|---|
| 섹터맵 함대 마커 **색** | `FUN_004ef0d0` | 사령관 char-table 엔트리 `+0xa/+0xb` 동등성(아=0x800/적=0x1000). 권위=0x0323 power@0x04 | **아니오** ([[logh7-faction-projection-2026-06-26]]) |
| 베이스 패널 **소속 진영명** | `FUN_0057aa90` | 진영명 **값**은 `0x031f` 동적 레코드 `elem+0x04` 직접 읽음. (이 함수는 `FUN_00522010`을 group **0x5f**/0x4e로 호출하지만 그건 정적 **필드 라벨** 텍스트이지 진영명 값이 아님) | **아니오** (group-1 미사용, [[base-panel-re-report]]) |
| 캐릭터 정보 패널 진영명 | `FUN_00597b20` | `_DAT_02227f68` low byte → `FUN_00522010(1,·)` | **예** |
| 게임메뉴 고정 진영 라벨 | `FUN_0059df00` | 리터럴 subId | **예(고정)** |

**결론**: group-1 constmsg 진영명은 **전략 섹터맵의 유닛/성계 표시 경로에 쓰이지 않는다**. 맵은 진영을
**색(FUN_004ef0d0)**으로, 베이스 패널은 **동적 0x031f 레코드(FUN_0057aa90)**로 진영을 표현한다. group-1
이름 텍스트의 소비처는 (a) 캐릭터 정보 패널 `FUN_00597b20`, (b) 게임메뉴 고정 라벨 `FUN_0059df00` 2곳뿐이다.

## 4. 불확실성/범위

- 본 xref는 **group 인자 리터럴 1** 한정이다. group을 변수로 받는 일반 디스패처(예: group 0x18 시스템명
  래퍼군 `FUN_004c8c90..`)는 진영 전용이 아니며 본 표에 포함하지 않았다 — 진영명 조회는 관례상 group을
  리터럴 1로 박으므로 이 필터가 진영명 소비처 전수에 적합하다.
- subId 매핑(`2=제국,3=동맹`)은 character-creation-wire 문서 기준의 기존 확정값을 인용한 것이며 본 사이클의
  신규 측정이 아니다.
- 라이브 미실행 사이클(정적 RE). 화면 확인은 별도 라이브 표면에서.
