## 도메인별 인벤토리

### 1. 화면 해상도 / 디스플레이

| 항목 | 근거 |
|---|---|
| 렌더 API | **Direct3D 8** — `g7mtclient.exe` PE 임포트에 `d3d8.dll`. 부수: `DINPUT8`, `DSOUND`, `WINMM`, `MSACM32`, `GDI32`, `IMM32`(일본어 IME) |
| 과거 기준 해상도 | 1024×768 (run9 이전 라이브 QA) |
| run9 로그인 | 내부 644×484 유지. 로그인 확대용 `login-native-layout` 33개 제거 |
| run9 로그인 후 | 1920×1080 네이티브 레이아웃, 창 캡처 1924×1084. `postlogin` 59개 = `lobby-res` 8 + layout 13 + `charsel` 38 |
| EXE 내 해상도 상수 | `1024`&`768` dword 쌍 81곳, **`1280`&`1024` dword 쌍 85곳**, `640`&`480` 5곳 → 단일 하드코딩이 아니라 **해상도 모드 테이블** 존재 시사. 1280×1024가 잠재/선택 가능 모드일 수 있음(RE 확인 필요) |
| 좌표·hitbox 의존성 | UI 레이아웃이 `exe/window2.dat`·`window3.dat`(각 4244B, 최근 라이브 세션이 생성)와 EXE `.rsrc` 다이얼로그 템플릿에 스크린 좌표로 존재 추정 |

**리마스터 실현가능성: 중.** run9로 로그인 후 1920×1080 레이아웃 경로는 입증됐다. 다만 이는 고해상도 texture·업스케일 완료가 아니며, 화면별 UI/hitbox 회귀와 자산 교체 검증은 별도다.

### 2. 모델링 (3D)

**3D 모델 존재함.** `data/model/` = **406 MDX**(독자 포맷, 31MB) + 12 MDS + 921 BMP 텍스처.

| 서브트리 | MDX 개수 | 비고 |
|---|---|---|
| `ship/fp` | 111 | 자유행성동맹(Free Planets) 함선 |
| `ship/ge` | 117 | 은하제국(Galactic Empire) 함선 |
| `ship/pi` | 18 | (pirates?) |
| `ship/pz` | 3 | (phezzan?) |
| `ship/` (루트) | 12 | 공통(`e_top.mdx` 등) |
| `planets` | 107 | 행성 3D |
| `space` | 8 | 우주 배경 |
| `effect` | 15 | 이펙트 지오메트리 |
| `light` | 6 | 라이트 |
| `strategy` | 7 | 전략맵 3D |
| `demo` | 1 | `opening.mdx` |

**함선 모델 합계 261, 전체 406.** MDX 헤더는 베이크된 포인터 테이블(base `0x01d00000`)과 명명 레이어(`black:Layer1`) 구조. MDS도 동일 포인터 구조(스켈레톤/서브모델 추정). **디코더 없음** — 기존 도구는 TCF 초상/이미지 경험만 있음.

**리마스터 실현가능성: 하~중.** 진짜 3D라서 고해상도화 여지는 크나, **MDX/MDS 포맷 RE가 선행 조건**. 포맷 해독 없이는 지오메트리·텍스처 참조 재패킹 불가. 이 도메인이 텍스처·이펙트 리마스터의 병목.

### 3. 텍스처

| 자산 | 위치 | 포맷 | 개수 | 해상도 분포 |
|---|---|---|---|---|
| 모델 텍스처 | `data/model/**/*.bmp` | BMP | 921 | 512×512(156), 256×256(138), 128×128(140), 512×256(98), 256×128(87), 128×64(78) — 전부 2의 거듭제곱 |
| 모델 TGA | `data/model/**/*.tga` | TGA | 10 | 16×8 ~ 1024×768 혼재 |

**리마스터 실현가능성: 상(디코드)/중(적용).** BMP/TGA는 표준 포맷이라 추출·업스케일 자명(512→2048 AI 업스케일 등). 단 텍스처 참조가 MDX 안에 있으면 재패킹이 **도메인2(MDX RE)에 종속**. 별도 텍스처 폴더 참조 방식이면 상.

## 리마스터 착수 우선순위 제안

원칙: **hitbox/좌표 안 깨는 것 · 원본 훼손 위험 낮고 효과 큰 것부터.**

1. **[최우선] 2D UI·초상·아이콘·메뉴 업스케일** — 도메인4. 논리 크기 유지 시 좌표 무손상. TCF 디코더 이미 보유(초상 1061). 실현가능성 상, 위험 하.
2. **전략맵·이벤트 CG** — 도메인4. `spot/`은 이미 1600×1200(그대로 활용), `strategy/` 512×512 업스케일.
3. **모델 텍스처 업스케일** — 도메인3. 품질 이득 크나 **MDX 텍스처참조 재패킹 필요** → 도메인2에 종속.
4. **1080p 화면별 회귀와 해상도 모드 테이블 조사** — 도메인1. 현재 1920×1080 경로를 기준으로 UI/hitbox를 회귀하고, 1280×1024 상수는 호환 모드 후보로만 RE한다.
5. **이펙트 텍스처** — 도메인5. MDX RE 이후.
6. **MDX/MDS 3D 포맷 RE** — 도메인2. 즉효는 낮으나 3·5번의 **enabler**. 함선/행성 지오메트리 리마스터의 전제.
7. **사운드** — 도메인6. 최하. 캐논 실황 보존 우선.
