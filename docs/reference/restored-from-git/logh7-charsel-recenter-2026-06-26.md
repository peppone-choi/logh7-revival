# charsel 캐릭생성 UI 1920 센터링 — RE + 재계산 + 후보패치 (2026-06-26)

## 1. RE: charsel 앵커 체계 (FUN_0051e580, 캐릭 생성/선택 8단계 폼 컨테이너)
- **함수**: FUN_0051e580 (호출자 FUN_0051ca30 씬 셋업, +0xc에 씬객체). 기존 패치가 적던 0x0051e589는 이 함수의 **엔트리 frame-rect 분배 코드**일 뿐 행좌표 아님.
- **패널 컨테이너 앵커 = 단 1점**: `FUN_00507a50(&local_2c8)` 에 전달되는 `local_2c8=300(0x12c)` X, `local_2c4=134(0x86)` Y.
  - foff `0x11e94e` (`c74424302c010000`, [esp+0x30]=300), foff `0x11e956` (`c744243486000000`, [esp+0x34]=134).
  - FUN_00507a50 = 씬객체 +0xc/+0x10에 X/Y 저장(순수 앵커 setter).
- **패널 크기 711x520** = `local_2d8=0x2c7(711)` / `local_2d4=0x208(520)` (foff 0x11e928 / 0x11e930). 불변.
- **자식 위젯 좌표 = 패널-로컬**: 각 위젯 rect(local_2e4/2e0/2d8/2d4)를 `FUN_00502940`가 widget+0x20에 **부모오프셋 미가산 그대로 복사**. 행 테이블 local_dc[]도 X=table+0x98, Y=table+0x67 식의 패널-내부 좌표. → **앵커 1점 이동이 전체 패널(진영선택·초상화 4슬롯·이름·능력치 행)을 통째로 옮김** (lobby-native-layout과 동일 아키텍처).

## 2. 원좌표 기준 = NATIVE display canvas (640 아님)
- 패널 폭 711 > 640 → 640x480 캔버스 불가. login/lobby native-layout과 동일 scene-anchor 계열. 즉 **3x 스케일은 의미 없음**; 센터=(캔버스−폭)/2.

## 3. 센터링 재계산 (1920x1080)

- 화면 내 확인: 604+711=1315<1920 ✓ / 280+520=800<1080 ✓.
