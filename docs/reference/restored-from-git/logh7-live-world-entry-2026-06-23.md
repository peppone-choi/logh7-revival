## 결과: 풀 라이브 월드진입 + 전략맵 렌더 (증거 보유)

- **트레이스 체인**(서버→클라 수신·소비): `0x7000`(로그인)→`0x0020`(로비)→`0x2000/2001/2006/2009/200a`(세션)→`0x0200/0x0201`(SS접속)→`0x0300..0x031f`(정적/정보)→`0x0313`×2/`0x0315`(전략그리드/섹터)→`0x0323`×2/`0x0325`×2(캐릭/유닛)→`0x0b09/0x0b0a`(그리드진입)→`0x0f02`(월드초기화)→`0x0f06/0x0f07`(재동기).
- **렌더 확인**(스크린샷): 전략 갤럭시 맵 — **다색 항성**(청색/주황/적색, RE의 O/B/A/F/G/K/M 다색 분포 라이브 입증·"파란 항성 하나" 반증 재확인), **100×50 전략 그리드**, **HUD**(좌하 사령관 초상화+상태바·중앙 미니맵·우하 한글 정보패널). 한글 로컬라이즈 라이브 정상.
- **stop SHA 복원**: shaVerified:true (canonical c1523a5e 복원). 잔류 프로세스 정리.

## #8 "월드진입 블로커"의 실제 정체 (증거 기반, 다중 원인)

그동안 "create-character 좌표 블로커"로 알려졌으나, 라이브 격리 결과 복합 원인:

1. **포그라운드 의존 스플래시(진짜 1차 원인)**: D3D8 클라의 부트/스플래시 루프는 **창이 포그라운드가 아니면 진행되지 않는다**. 포그라운드 미강제 시 BOTHTEC 스플래시에서 무한 정체(접속 0건, 프레임 동결). **PowerShell `SetForegroundWindow`를 부팅 ~35초 동안 반복 유지**하면 스플래시를 지나 로그인/월드로 진행. (2026-06-22 전 라이브 세션이 모두 2줄 trace로 실패한 근본 — 코드/EXE 아님.)
2. **부트스트랩 제거로 자동로그인 부재**: canonical c1523a5e는 `login-commandline-bootstrap`(741b, OFF) 제거됨(2026-06-22 사용자 결정 "테스트 id 없애기"). 수동 폼은 빈 제출 시 ginei00 폴백으로 접속 가능하나(login-blank-password-local-ok), **D3D8 입력 미반영(아래)** 때문에 수동 클릭/타이핑이 안 먹혀 막힘.
3. **D3D8 입력 인젝션 미반영(잔존 프런티어)**: 로그인 폼/버튼에 **마우스 클릭·키보드 입력이 등록되지 않음**. `text`=WM_CHAR는 무시(필드는 DirectInput/폴링), `key --hw` keybd_event도 별도프로세스 포그라운드 절도로 실패. 메모리/스킬의 "마우스 미해결(cursor-clip/DirectInput)"과 정합.
4. **하네스 좌표 해상도 불일치**: `run_login_flow` 로그인 좌표(325,333~389)는 640×480 기준인데 실제 렌더는 1920×1080 → 빗나감. 세션/창작 좌표는 일부 HD(880,343).

**해법(현행)**: autologin 변종(부트스트랩 내장) + 포그라운드 유지 → 무클릭 월드진입. **4클라 E2E도 동일 경로**(emp1/emp2/all1/all2 변종).
