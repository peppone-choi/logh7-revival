# charsel 배경↔내용 어긋남 — 부분 recenter 버그 진단·수정 (2026-06-26)

## 증상 (사용자 라이브 보고)
캐릭터 생성 화면에서 **배경 패널은 화면 중앙으로 옮겨졌는데 내용(진영 라디오 / 이름칸 / 능력치 / 초상화) 좌표는 안 따라옴** = 배경↔내용 어긋남. "클릭 좌표(드라이버) 문제가 아니라" EXE 렌더 정합 문제.

## 근본 원인 (확정)
`charsel-recenter.json` 패치가 **단 2바이트 오프셋만** 이동:
- foff `0x11e94e`: 앵커 X `300(0x12c) → 604(0x25c)`
- foff `0x11e956`: 앵커 Y `134(0x86) → 280(0x118)`
- 둘 다 **FUN_0051e580** (1개 패널). 델타 = **(+304, +146)**.

그러나 캐릭터 생성 8단계 폼의 **실제 내용**은 **별도 패널 FUN_0051f8b0**(자체 앵커 (300,134) + 서브앵커 (605,206))과 FUN_0051ca30 씬 셋업이 호출하는 나머지 빌더들에 있다. 이들 앵커는 **패치 안 됨** → FUN_0051e580(배경/프레임)만 중앙으로 가고 내용은 네이티브 (300,134)에 잔류.

원래 charsel-recenter.json의 가정 "자식 위젯은 패널-로컬이라 앵커 1점 이동이 전체를 옮긴다"는 **라이브로 반증**됨 — 화면은 단일 패널이 아니라 FUN_0051ca30이 조립하는 **다중 형제 패널**이고, 각자 독립 앵커를 가진다(login-native-layout "640 트랩"과 동형 = 프레임만 옮기고 컨트롤은 잔류).

## 수정 방향
charsel 화면을 **코히런트하게** recenter하려면 FUN_0051ca30 빌더 체인의 **내용 앵커들도 같은 델타 (+304, +146)** 로 이동해야 한다. 단:
- 각 즉치값이 정말 **앵커 좌표**(setter 인자)인지 vs 폭/높이/rect 필드인지 setter 호출 인자 위치로 확증(640 트랩 재발 방지).
- 서브앵커 (605,206)이 **절대좌표**면 +304/+146, **상대(부모+오프셋)**면 건드리면 이중이동 → 건드리지 않음.
