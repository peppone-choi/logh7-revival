# LOGH VII 오리지널 캐릭터 포트레잇 네이밍 툴

은하영웅전설 VII의 **오리지널(원작 캐릭터) 포트레잇 446장**(O군: `oem`=제국 187, `oam`=동맹 181, `o`=기타 78)에
이름을 다는 독립 실행 도구입니다. **편집하는 즉시 `names.json`에 자동 저장**됩니다(별도 저장 버튼 불필요).

> **같은 얼굴 자동 묶음**: 446장 중 상당수가 바이트 단위로 동일한 중복입니다. 이 툴은 중복을 묶어
> **유니크한 얼굴 289개**만 보여주고, **한 번 이름을 달면 같은 얼굴 전부에 적용**됩니다(중복 카드 우상단 `×N` 배지).

## 실행

1. **`start.bat` 더블클릭** → 콘솔 창이 뜨고 브라우저가 자동으로 열립니다.
2. 안 열리면 브라우저에서 `http://127.0.0.1:8799/` 직접 입력(콘솔 창에 주소가 표시됨).
3. 종료: 콘솔 창에서 **Ctrl+C** (모든 변경은 이미 저장되어 있음).

> Python 3 필요(없으면 https://www.python.org). 인터넷 연결 불필요 — 전부 내 PC 안에서만 동작.

## 사용법

- 각 카드 = **유니크한 얼굴 1개**. **이름(한글)** 칸에 입력하면 0.3초 뒤 자동 저장되고 `저장됨 HH:MM:SS`가 뜹니다.
  중복(`×N`)이면 그 이름이 같은 얼굴의 모든 사본에 함께 적용됩니다.
- **자동완성**: 이름 칸 입력 시 정규 캐릭터명(한글/일본어) 후보가 뜹니다(`suggestions.json`, 직접 편집 가능).
- **진영 / 일본어 원명 / 메모**는 선택 입력. 진영은 아틀라스(oem→제국, oam→동맹)에 따라 기본값이 잡혀 있습니다.
- **Enter** 키: 다음 카드의 이름 칸으로 이동(빠른 연속 명명).
- 상단 도구막대: 검색, 진영(아틀라스) 필터, `미지정/지정됨/중복만` 필터, 진행률(명명 얼굴/289), **백업** 버튼.

## 저장 파일

- **`names.json`** — 실시간 자동저장되는 레지스트리. **얼굴(중복 그룹) 단위**로 저장됩니다:
  ```json
  {
    "entries": {
      "<face-id>": { "name":"라인하르트", "name_ja":"", "faction":"empire", "note":"",
                     "rep":"oem/0112.png", "members":["oem/0112.png", "..."], "count":3, "updatedAt":"..." }
    },
    "savedAt":"YYYY-MM-DD HH:MM:SS"
  }
  ```
  `<face-id>` = 이미지 내용 해시(같은 얼굴 = 같은 id). `members` = 그 얼굴의 모든 이미지 경로.
- **이미지별로 펼친 형태**가 필요하면 `http://127.0.0.1:8799/api/export` (각 이미지 경로 → 이름).
- 저장은 **원자적**(tmp 후 교체)이라 작업 중 꺼져도 파일이 깨지지 않습니다.
- **백업** 버튼 → `names.backup-YYYYMMDD-HHMMSS.json`. 전체 상태 다운로드: `/names.json`.
- 예전 버전의 이미지별 `names.json`도 실행 시 **자동으로 얼굴 단위로 마이그레이션**됩니다.

## 구성

```
LOGH7-OriginalPortraitNamer/
  start.bat          ← 더블클릭해서 실행 (순수 ASCII; 한글 X — cmd 파싱 보호)
  serve.py           ← 로컬 서버(표준 라이브러리만 사용; 중복 그룹화 포함)
  index.html         ← 브라우저 GUI
  suggestions.json   ← 자동완성 후보(편집 가능)
  names.json         ← 결과(실행 후 자동 생성·갱신)
  portraits/{o,oam,oem}/*.png  ← 포트레잇 446장
```

데이터 출처: `content/roster/canon-portraits/`. 기존 `canon-face-registry.json`의 faceCode 매핑은
신뢰 불가(`_WARNING`)라 이 툴은 **사람이 이미지를 보고 직접 재명명**하는 것을 전제로 합니다.
재빌드/재배포: 저장소에서 `python tools/standalone/original-portrait-namer/build_desktop.py`.
