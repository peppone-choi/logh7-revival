# .ai/ — 세션 동적 상태 (정본)

이 디렉터리는 **현재 작업 세션의 동적 상태 정본**이다.
영구 규칙은 `CLAUDE.md`·`AGENTS.md`·`docs/agent/`, 장기 실행계획은 `.omo/plans/`,
영구 지식은 `docs/`에 있다. 여기에는 "지금 무엇을 하고 있고, 무엇이 결정됐고,
누가 어떤 파일을 소유하는가"만 둔다. (계층 근거: `.ai/decisions.md` ADR-LITE-004)

| 파일 | 역할 | 갱신 주체 |
|---|---|---|
| `task.md` | 현재 작업 계약 (목표·범위·수용 기준) | 사람이 작성 또는 승인 |
| `current-state.md` | 진행 상태 압축 스냅샷 (항상 1개) | 에이전트 |
| `decisions.md` | 인간 승인 결정 (ADR-LITE) | 에이전트가 기록, 사람이 승인 |
| `known-issues.md` | 알려진 미해결 문제 | 에이전트 |
| `ownership.md` | 다중 에이전트 파일 소유권 | 각 에이전트 (작업 시작·종료 시) |
| `handoff.md` | 다음 세션·다른 에이전트 인수인계 | 에이전트 (세션 종료 전) |

## 규칙

- `current-state.md`는 로그 저장소가 아니다 — 항상 최신 스냅샷 하나로 덮어쓴다.
- `decisions.md`의 `approved` 상태는 사람만 부여한다. 에이전트는 `proposed`까지만 쓴다.
- `task.md`가 비어 있거나 `Status: EMPTY`면, 구현을 시작하기 전에 사람에게 작업 계약을 확인받는다.
- 오래된 상태를 그대로 믿지 않는다: `Updated at`이 현재 HEAD·브랜치와 맞지 않으면 stale로 간주하고 재확인한다.
- `.omc/`·`.omo/` 등 플러그인 런타임 상태는 이 계층과 별개다. 여기 파일이 우선한다.
