# /checkpoint

$ARGUMENTS: (선택) 체크포인트 사유 — 컨텍스트 리셋 전, 인수인계 전, 주요 단계 완료 후.

절차:

1. `.ai/current-state.md` 갱신 — 완료/진행/변경 파일/검증 결과/실패한 접근/다음 행동. 로그 누적 금지, 최신 스냅샷 하나로 덮어쓴다.
2. 인수인계가 필요하면 `.ai/handoff.md` 갱신 — `docs/agent/prompt-pack.md`의 "작업 인수인계" 필수 항목 준수 (결정과 추측 분리, 실행한/안 한 검증 분리, 실패한 접근, 다음에 읽을 파일).
3. `.ai/ownership.md` 갱신 — 끝난 작업은 done 처리, 진행 중이면 Updated at 갱신.
4. 이번 세션에서 사람이 승인한 결정이 있으면 `.ai/decisions.md`에 추가한다 (approved는 사람 승인이 있을 때만).

완료 조건: 다음 세션이 대화 기록 없이 `.ai/` 파일만 읽고 작업을 재개할 수 있는 상태.
